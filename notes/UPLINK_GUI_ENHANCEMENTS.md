## 📌 개요

현재 uplinkGUI는 기본 기능(CONFIG, ROUTE, RECOVERY)을 제공하고 있습니다.  
다음 단계에서는 **사용성 개선**과 **안전성 강화**를 목표로 6가지 기능을 추가합니다.

### 🔴 2주차 추가 구현

#### ① 명령 히스토리 + 재실행 (1.5시간)

**필요성**: 같은 설정을 반복할 때 다시 입력 불필요

**기능 명세**:
```
- 최근 10개 명령 자동 저장 (localStorage)
- 각 명령 항목: scope, param, value, timestamp, status
- 클릭하면 폼에 자동 로드
- "이전과 같이" 버튼으로 빠른 재실행
```

**UI**:
```
[최근 명령]

 2. config mavlink_bridge attitude_interval_us 50000  [1시간 전] [✅]
 3. recovery                                   [1일 전] [✅]

[클릭] → 폼에 자동 입력 + "다시 전송" 가능
```

#### ② 설정 템플릿 저장/로드 (2시간)

```

- 템플릿 이름 지정 (예: "고속 비행", "저전력", "로깅 모드")

- 템플릿 삭제 기능
```

```json
{

      "created": "2026-07-09T14:30:22"
    },
    {
      "name": "저전력",
      "commands": [
        {"scope": "cfs_core", "param": "publish_period_ms", "value": 500}

    }
  ]
}

**UI**:
```

 ▼ 저전력

**구현 위치**:

#### ③ 명령 큐 + 일괄 전송 (2.5시간)

**필요성**: 여러 설정을 한번에 순차 전송 (자동 응답 대기)

**기능 명세**:
```
- 명령을 "큐"에 추가 (아직 전송 안 함)
- 큐에 여러 명령 쌓기 가능
- "모두 전송" 버튼 클릭 시 순차 실행
- 각 명령마다 downlink 응답 대기 후 다음 명령 전송
- 큐 항목별 상태: 대기 중 / 전송 중 / 완료 / 실패
```

**UI**:
```
[명령 큐]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 ☑ [1] config cfs_core attitude_timeout_ms 1000        [⏳ 대기]
 ☑ [2] config mavlink_bridge attitude_interval_us 40000 [⏳ 대기]
 ☑ [3] config cfs_core publish_period_ms 100           [⏳ 대기]
 ☐ [4] recovery                                         [⏳ 대기]

 [▶ 모두 전송 (3개)] [❌ 초기화] [↑/↓ 순서변경]
```

**실행 시나리오**:
```
사용자 [▶ 모두 전송] 클릭
  ↓
[1] 전송 → 상태 변경 [⏳ 전송 중]
  ↓ (downlink 응답 대기)
  ↓ (UFB=0 확인) → 상태 변경 [✅ 완료]
  ↓
[2] 전송 → 상태 변경 [⏳ 전송 중]
  ↓ ...
  ↓
[3] 전송 + 완료
  ↓
[4] 스킵 (☐ 체크 해제)
  ↓
모두 완료!
```

**구현 위치**:
- `plugin.js`: 큐 관리, 순차 실행 로직
- `form.css`: 큐 UI, 상태 표시

**검증 기준**:
- [ ] 명령 추가/삭제/순서변경 동작
- [ ] 큐에서 선택된 항목만 전송
- [ ] 각 명령 사이에 downlink 응답 대기
- [ ] 응답 시간초과 시 다음 명령 진행 또는 중단
- [ ] 중도 취소 가능

#### ④ Uplink Feedback 모니터링 (1.5시간)

**필요성**: 명령이 정말 기체에서 적용되었는지 자동으로 확인

**기능 명세**:
```
- 명령 전송 시 sequence 번호 기억
- downlink 수신 시 UFB(Uplink Feedback) 필드 확인
- UFB 값에 따라 자동 처리:
  * UFB=0 (OK): 성공 표시
  * UFB=1 (CRC_FAIL): 자동 재전송
  * UFB=2 (SEQ_FAIL): 경고 + 다시 시도 버튼
```

**UFB 값 정의**:
```
| UFB | 의미 | 처리 |
|-----|------|------|
| 0 | OK ✅ | 초록색 "적용됨" |
| 1 | CRC_FAIL | 빨강 "패킷 손상, 재전송..." |
| 2 | SEQ_FAIL | 주황 "시퀀스 오류, 다시 시도" |
```

**로그 예시**:
```
[OK] CONFIG accepted  seq=5  cfs_core.publish_period_ms=100
     ↓ (100ms, downlink 수신)
[✅ UFB=0] 기체가 명령을 수신했습니다!
    cfs_core.publish_period_ms = 100ms (적용됨)

---

[OK] CONFIG accepted  seq=6  mavlink_bridge.attitude_interval_us=40000
     ↓ (100ms, downlink 수신)
[❌ UFB=1] CRC 오류! 자동으로 다시 전송합니다...
     ↓ (100ms, downlink 수신)
[✅ UFB=0] 재전송 성공!
    mavlink_bridge.attitude_interval_us = 40000us (적용됨)
```

**구현 위치**:
- `plugin.js`: downlink 모니터링, UFB 파싱
- cfsRealtime 플러그인과의 통신 (downlink 피드

)

**검증 기준**:
- [ ] UFB 필드 정확히 파싱됨
- [ ] 각 UFB 값에 대해 올바른 처리
- [ ] 자동 재전송 동작
- [ ] 로그에 시간 기록 (명령 → UFB 응답)

### 🟡 3주차 추가 구현

#### ⑤ 설정값 범위 제안 (1시간)

**필요성**: "이 파라미터는 50~5000ms 범위입니다" 같은 가이드 제공

**기능 명세**:
```
- 서버(/api/uplink/meta)에서 메타데이터 조회
- 각 파라미터별 min/max/recommended 값 받기
- 입력 시 범위 체크 + 경고
- 추천값 자동 채우기 옵션
```

**메타데이터 형식** (fc_serial_ws_server.py에서 제공):
```json
{
  "cfs_core": {
    "attitude_timeout_ms": {
      "type": "uint32",
      "min": 100,
      "max": 10000,
      "recommended": 1000,
      "description": "FC 자세 정보 타임아웃"
    }
  }
}
```

**UI**:
```
[파라미터 선택] publish_period_ms ▼
📋 범위: 50 ~ 5000 ms (권장: 100)
[값] [_________]

입력값이 범위 초과 시:
⚠️ "5000보다 큰 값은 권장하지 않습니다"
```

**구현 위치**:
- `plugin.js`: 메타데이터 조회 및 검증
- `form.css`: 범위 표시 UI

**검증 기준**:
- [ ] 메타데이터 조회 동작
- [ ] 범위 검증 로직
- [ ] 범위 초과 시 경고 표시
- [ ] 추천값 자동 채우기 (선택)

#### ⑥ 롤백 기능 (2시간)

**필요성**: 설정 변경 후 문제 생기면 이전으로 돌아가기

**기능 명세**:
```
- 각 명령 전송 전에 현재 상태 스냅샷 저장
- 스냅샷 목록 표시 (타임스탬프, 변경사항)
- "이 시점으로 복구" 클릭 → 이전 설정값들 자동 재전송
- 롤백 전 확인 대화
```

**스냅샷 저장 구조**:
```json
{
  "snapshots": [
    {
      "timestamp": "2026-07-09T14:30:22",
      "changes": [
        {
          "scope": "cfs_core",
          "param": "publish_period_ms",
          "old_value": 100,
          "new_value": 50
        }
      ],
      "status": "completed"
    },
    {
      "timestamp": "2026-07-09T14:25:00",
      "changes": [...]
    }
  ]
}
```

**UI**:
```
[설정 스냅샷 (변경 이력)]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 2026-07-09 14:30:22 (3개 변경)
   - cfs_core.publish_period_ms: 100 → 50 ❌ (문제 발생)
   - mavlink_bridge.attitude_interval_us: 40000 → 20000
   - cfs_core.attitude_timeout_ms: 1000 → 500
   [↩️ 이 시점으로 복구]

 2026-07-09 14:25:00 (1개 변경)
   - cfs_core.publish_period_ms: 100 → 100
   [↩️ 이 시점으로 복구]
```

**구현 위치**:
- `plugin.js`: 스냅샷 저장/복구 로직, 변경사항 추적
- `form.css`: 스냅샷 목록 UI

**검증 기준**:
